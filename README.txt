# Курс DARALTAI — публикация на GitHub Pages

В папке уже находится готовый статический сайт. Для работы нужны все файлы и папка `assets`.

## Состав проекта

- `index.html` — главная страница курса;
- `01_podgotovka.html` — подготовка;
- `02_contact.html` — установление контакта;
- `03_potrebnosti.html` — выявление потребностей;
- `04_presentation.html` — презентация;
- `05_vozrazheniya.html` — работа с возражениями;
- `06_sdelka.html` — завершение сделки;
- `07_itog.html` — итоговый урок;
- `assets/` — логотип, стили и скрипты;
- `.nojekyll` — отключает обработку сайта через Jekyll.

## Вариант 1. Загрузка через сайт GitHub

1. Войдите в GitHub.
2. Нажмите значок `+` в правом верхнем углу и выберите **New repository**.
3. Укажите название репозитория, например `daraltai-course`.
4. Выберите **Public**.
5. Не добавляйте README, .gitignore и лицензию, так как готовые файлы уже есть в архиве.
6. Нажмите **Create repository**.
7. На странице репозитория нажмите **Add file → Upload files**.
8. Распакуйте архив на компьютере.
9. Перетащите в окно GitHub всё содержимое распакованной папки, а не сам ZIP-файл. В корне репозитория должен сразу находиться `index.html`.
10. Проверьте, что папка `assets` загрузилась как папка и внутри неё видны файлы.
11. Внизу страницы в поле сообщения можно написать `Загрузка курса DARALTAI`.
12. Нажмите **Commit changes**.
13. Откройте **Settings → Pages**.
14. В разделе **Build and deployment** в поле **Source** выберите **Deploy from a branch**.
15. В поле **Branch** выберите `main`, рядом выберите папку `/(root)` и нажмите **Save**.
16. Подождите несколько минут и обновите страницу **Settings → Pages**. GitHub покажет адрес опубликованного сайта.

Адрес проекта обычно выглядит так:

`https://ИМЯ-ПОЛЬЗОВАТЕЛЯ.github.io/daraltai-course/`

## Вариант 2. Сайт по адресу без названия репозитория

Чтобы сайт открывался по адресу `https://ИМЯ-ПОЛЬЗОВАТЕЛЯ.github.io/`, назовите репозиторий строго:

`ИМЯ-ПОЛЬЗОВАТЕЛЯ.github.io`

Остальные действия такие же.

## Как обновлять курс

1. Откройте репозиторий.
2. Нажмите **Add file → Upload files**.
3. Перетащите обновлённые файлы с теми же именами.
4. Подтвердите замену и нажмите **Commit changes**.
5. GitHub Pages автоматически опубликует новую версию после обработки изменений.

## Проверка после публикации

- главная страница открывается;
- отображается логотип DARALTAI;
- кнопка «Начать курс» ведёт на первый урок;
- работают переходы «Предыдущий урок» и «Следующий урок»;
- стили и логотип загружаются;
- после обновления страницы прогресс курса сохраняется в том же браузере.

## Частые ошибки

### Открывается страница 404

Проверьте:

- включён ли GitHub Pages в **Settings → Pages**;
- выбраны ли ветка `main` и папка `/(root)`;
- находится ли `index.html` в корне репозитория, а не внутри дополнительной папки;
- прошло ли несколько минут после включения Pages.

### Нет оформления или логотипа

Проверьте, что папка `assets` загружена целиком и её имя написано строчными буквами.

### Загрузился ZIP, но сайт не работает

GitHub Pages не распаковывает архив. ZIP нужно распаковать на компьютере и загрузить файлы из него.

### После обновления видна старая версия

Откройте сайт в режиме инкогнито или выполните жёсткое обновление страницы: `Ctrl + Shift + R` на Windows либо `Command + Shift + R` на macOS.
